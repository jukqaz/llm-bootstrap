# Autopilot

Use this when a task needs an end-to-end pass instead of a single reply.

## Sequence

1. Confirm the target outcome.
2. Plan the smallest safe implementation.
3. Execute bounded slices.
4. Re-run QA until the critical path is proven or a repeated issue blocks progress.
5. End with review and acceptance checks.

## When to stop

- The same failure repeats three times.
- The next decision is destructive or preference-driven.
- The task is no longer a build task and has become open-ended research.
