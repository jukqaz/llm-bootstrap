# PRD

## 요약
- 한 줄 목표: 개인용 Codex 스킬을 관리하고 확장하기 위한 저장소를 유지한다.
- 문제 정의: 스킬 규칙과 문서가 분산되면 관리와 재사용이 어려워진다.

## 목표
- 스킬 구조를 표준화하고 재사용성을 높인다.
- AGENTS/PRD/TRD로 문서 구조를 일관되게 유지한다.
- 스킬 추가 및 갱신 흐름을 간단하게 만든다.

## 비목표
- 실행 가능한 서비스나 앱을 제공하지 않는다.
- 배포 파이프라인 구축을 포함하지 않는다.

## 사용자 및 사용 사례
- 사용자: 저장소 소유자
- 사용 사례:
  - 새 스킬 추가
  - 기존 스킬 업데이트 및 정리
  - 템플릿 재사용

## 요구사항
### 기능
- `skills/` 아래에 스킬 디렉터리를 유지한다.
- 각 스킬은 `SKILL.md`를 포함한다.
- 문서 링크는 `AGENTS.md`에 명시한다.

### 비기능
- 문서는 한국어로 작성한다.
- 파일명은 대소문자를 정확히 유지한다.
- 내용은 짧고 명확하게 유지한다.

## UX 및 흐름
- 스킬 추가: 스킬 디렉터리 생성 → `SKILL.md` 작성 → 필요 리소스 추가 → 문서 링크 갱신.

## 성공 지표
- 스킬 추가 시 재사용 가능한 템플릿이 존재한다.
- 문서/링크 불일치가 발생하지 않는다.

## 의존성
- Codex CLI
- Git

## 리스크 및 가정
- 리스크: 문서 갱신 누락으로 규칙 불일치 발생.
- 가정: 저장소는 개인용으로 운영된다.
