# PRD/TRD 질문 목록

AGENTS.md와 README.md를 읽은 뒤 누락된 정보만 질문하라.

## 범위 확인 (필수)
1) 어떤 문서를 생성/갱신해야 하나? (`PRD.md`, `TRD.md`)
2) 각 문서의 위치(경로)는 어디인가? 기본값은 리포 루트.
3) 네이밍/대소문자 확인: `PRD.md`, `TRD.md`를 정확히 사용하라.

## PRD (제품 요구사항, 필수)
1) 어떤 문제를 누구를 위해 해결하나?
2) 범위 포함/제외 항목은 무엇인가?
3) 성공 지표 또는 수용 기준은 무엇인가?
4) 핵심 사용자 흐름/UX 제약은?
5) 의존성이나 리스크는?

## TRD (기술 요구사항, 필수)
1) 상위 아키텍처와 핵심 컴포넌트는?
2) 데이터 모델 또는 저장소 선택은?
3) 인터페이스/API 또는 통합 지점은?
4) 실행/빌드/테스트/배포 명령과 환경은?
5) 소스관리 도구와 워크플로우는? (예: git, jj)
6) 보안/관측/운영 제약은?

## 기존 문서 (선택)
1) PRD/TRD가 있으면 반드시 유지해야 할 내용은?
2) 제거하거나 병합해야 할 섹션이 있는가?
