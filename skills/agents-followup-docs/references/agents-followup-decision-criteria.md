# 범위 결정 기준

문서 범위를 선택할 때 사용하라. 필수 정보를 담을 수 있는 최소 범위를 기본값으로 하라.

## 라이트 (Light)
다음이 모두 참이면 라이트를 선택하라:
- 단일 모듈 또는 단일 배포 대상.
- 외부 서비스/연동이 적음.
- 단기 작업 또는 낮은 리스크.

산출물:
- 핵심 섹션만 포함한 `PRD.md`와 `TRD.md`.

## 표준 (Standard)
다음 중 하나라도 해당되면 표준을 선택하라:
- 2~3개 모듈 또는 명확한 FE/BE 분리.
- 외부 서비스(인증/결제/분석 등) 1개 이상.
- 정기 배포 또는 지속 운영.

산출물:
- 범위/성공 지표를 포함한 `PRD.md`.
- 아키텍처, 인터페이스, 실행/테스트/배포 기본을 담은 `TRD.md`.

## 확장 (Expanded)
다음 중 하나라도 해당되면 확장을 선택하라:
- 모노레포 또는 4개 이상 모듈.
- 여러 팀 또는 복잡한 소유 경계.
- 컴플라이언스, 보안, 높은 운영 리스크.

산출물:
- 의존성, 마일스톤, 리스크 계획을 포함한 `PRD.md`.
- 상세 컴포넌트 분해와 통합 지점을 담은 `TRD.md`.
- 선택 사항: 모듈별 TRD 섹션 또는 ADR(요청 시에만 추가).

## Notes
- AGENTS.md가 특정 문서 경로를 이미 링크하면 그 경로를 따르라.
- 사용자가 분기 선택에 동의하지 않으면 사용자의 선택을 따르라.
