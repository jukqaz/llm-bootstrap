# 범위 결정 기준

필수 정보를 담을 수 있는 최소 범위를 기본값으로 하라.

## 라이트 (Light)
- 단일 모듈/단일 배포.
- 외부 연동이 적음.
- 단기 작업.

산출물:
- 핵심 섹션만 있는 PRD/TRD.

## 표준 (Standard)
- 2~3개 모듈 또는 FE/BE 분리.
- 외부 서비스 1개 이상.
- 지속 운영 가능성.

산출물:
- 요구사항과 성공 지표가 포함된 PRD.
- 아키텍처/인터페이스/실행 정보가 포함된 TRD.

## 확장 (Expanded)
- 모노레포 또는 4개 이상 모듈.
- 팀/책임 경계가 복잡함.
- 보안/운영 리스크가 높음.

산출물:
- 의존성/마일스톤/리스크 계획 포함 PRD.
- 상세 컴포넌트 분해 포함 TRD.
- 필요 시 모듈별 부록 문서.
