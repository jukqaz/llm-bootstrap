# GitHub PR + CI/CD

## Overview

GitHub PR 흐름과 CI/CD 상태를 체계적으로 점검하라. 기본은 GitHub MCP 사용이며, MCP가 없으면 git/gh 기반으로 진행하라. 모든 결과는 한국어로 요약하라.

## Workflow

### 1) 컨텍스트 수집
- `AGENTS.md`, `README.md`, `PRD.md`, `TRD.md`를 확인하라.
- 대상 리포/브랜치/PR 링크, 목표 브랜치, 변경 범위를 파악하라.
- 정보가 부족하면 먼저 질문하라.

### 2) 도구 선택
- GitHub MCP가 설정되어 있으면 우선 사용하라.
- MCP가 없으면 `gh` 또는 `git`으로 진행하되, 실행 전 사용 여부를 확인하라.

### 3) PR 처리
- PR이 있으면 상세/변경 파일/리뷰/체크 상태를 정리하라.
- PR이 없으면 제목/본문 초안을 만들고 생성 여부를 물어라.
- 문서 변경이 있으면 `docs-audit` 또는 `doc-linker` 사용을 검토하라.
- 릴리즈가 포함되면 `release-docs` 연계를 제안하라.

### 4) CI/CD 점검
- 체크 실패 여부와 실패 지점을 요약하라.
- 실패 로그를 수집해 원인을 분류하라(테스트/린트/빌드/배포/권한/환경).
- 수정안 또는 재시도 전략을 제시하라.

### 5) 재실행/업데이트
- 실패 원인 수정 후 재실행 여부를 확인하라.
- PR 본문/코멘트/리뷰 상태를 갱신하라.
- 필요한 경우 리뷰 요청자를 확인하라.

## MCP/도구 연계

- PR 상태: PR 상세/파일/리뷰/상태 확인
- CI/CD: 워크플로우 실행/잡/로그 확인, 실패 재실행
- PR 업데이트: 제목/본문/리뷰 요청
- 설정 여부가 불명확하면 먼저 확인하라.

## Output checklist

- PR 유무와 상태가 명확하다.
- 실패한 체크/잡과 원인이 요약되었다.
- 수정안 또는 재실행 전략이 제시되었다.
- 문서/릴리즈 연계가 검토되었다.
