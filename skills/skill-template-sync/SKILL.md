# 스킬 템플릿 동기화

## Overview

템플릿 파일을 비교해 차이를 리포트하고, 승인 후 동기화하라. 소스 템플릿과 대상 템플릿을 명확히 구분하라.

## Workflow

### 1) 템플릿 매핑 확인
- `references/template-map.md`에서 소스와 대상을 확인하라.
- 매핑이 없으면 사용자에게 소스/대상을 물어라.

### 2) 차이 분석
- 소스와 대상 템플릿을 비교하라.
- 의미 있는 차이(섹션/용어/순서)를 요약하라.

### 3) 동기화
- 승인 후 소스를 기준으로 대상 템플릿을 갱신하라.
- 파일명 대소문자는 변경하지 말아라.

### 4) 결과 공유
- 변경된 파일과 변경 요약을 제공하라.

## Output checklist

- 소스/대상 매핑이 명확하다.
- 변경 사항이 승인 후에만 반영되었다.
