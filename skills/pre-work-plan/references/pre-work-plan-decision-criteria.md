# 복잡도 분기 기준

최소 정보로도 명확하면 단순로 분류하라. 하나라도 해당되면 복잡으로 분류한다.

## 단순 (Simple)
- 단일 모듈/단일 영역 변경.
- 변경 파일 수가 적고 영향 범위가 좁음.
- 외부 연동, 데이터 변경, 배포 영향 없음.
- 리스크가 낮고 테스트 범위가 작음.

## 복잡 (Complex)
- 다중 모듈 또는 광범위 변경.
- 외부 연동, 데이터 마이그레이션, 배포 영향 포함.
- 테스트 범위가 넓거나 리스크가 중~높음.
- 이해관계자/의존성이 많음.

## Notes
- 판단이 모호하면 사용자에게 확인하라.
- 복잡 작업은 Linear 이슈로 관리하라.
