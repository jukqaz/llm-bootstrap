# AGENTS.md 질문 시퀀스

## 필수 질문
1. 응답 언어/톤은 무엇인가? (예: 한국어 고정, 간결한 톤)
2. 프로젝트 타입과 주요 기술 스택은 무엇인가? (예: Flutter/Dart, 웹, 백엔드)
3. 모노레포인가? 주요 모듈/서브프로젝트와 경로는 무엇인가?
4. 루트/모듈별 중첩 AGENTS.md를 둘 필요가 있는가? 있다면 어디에 둘까?
5. AGENTS.md에 반드시 포함할 섹션은 무엇인가?
6. 빌드/실행/테스트/코드생성 명령은 어떤 것들이 있는가? (요약+링크만 둘지 확인)
7. 모듈 간 교차 워크플로우(예: FE->BE API, 인증/세션, 공유 타입)는 무엇인가?
8. Git/브랜치/커밋/PR 규칙은 무엇인가?
9. 민감정보/보안/파괴적 명령에 대한 규칙은 무엇인가?
10. MCP/외부 도구 사용 규칙(허용/금지, 권한/토큰 처리)은 무엇인가?
11. 참조해야 할 문서는 무엇인가? (README/PRD/TRD 등)

## 선택 질문
1. 프로젝트 구조 요약을 AGENTS.md에 포함할까, 링크로만 둘까?
2. 모듈 맵(모노레포)이나 기능 맵(모듈별)을 추가할까?
3. 외부 서비스/환경설정 관련 주의사항을 넣을까?
4. 성능/테스트 전략(예: golden/patrol 등)을 넣을까?

## 기본 제안(사용자 응답 없을 때)
- AGENTS.md는 행동 규칙 위주로 유지하고 상세 기획/설계는 PRD/TRD로 분리.
- 빌드/테스트는 요약+링크만 유지.
- 커밋 메시지는 Conventional Commits 사용.
- 파괴적 명령은 반드시 사용자 확인 후 수행.
