# 제품/전략

## Overview

제품/전략 역할에서 필요한 산출물과 작업 흐름을 정리하라. 한국어로 작성하고 문서/작업 결과를 간결하게 요약하라.

범위: 제품 비전/포지셔닝/전략 방향, 로드맵 원칙 정리
비범위: PRD/TRD 상세 작성(project-planning-docs, agents-followup-docs), 기능 맵(feature-map-builder)

## Workflow

### 1) 컨텍스트 수집
- AGENTS.md, README.md, PRD.md, TRD.md를 확인하라.
- 목표, 범위, 제약, 일정 정보를 정리하라.
- 정보가 부족하면 먼저 질문하라.

### 2) 산출물 정의
- 필요한 산출물을 PRD, TRD, 로드맵, 기능/모듈 맵 기준으로 정리하라.
- 산출물 우선순위를 정하고 범위를 명확히 하라.

### 3) 세부 스킬 연계
- 필요 시 다음 스킬을 사용하라:
  - 기획 문서화: project-planning-docs
  - AGENTS 기반 문서 정리: agents-followup-docs, agents-md
  - 로드맵/기능 맵/요구사항 리뷰: roadmap-planner, feature-map-builder, requirements-review
- 외부 도구나 MCP 설정이 필요하면 먼저 확인하라.

### 4) 정리 및 공유
- 결과를 핵심 요약과 함께 공유하라.
- 관련 문서/링크를 갱신하라.

## Output checklist

- 산출물이 PRD/TRD/로드맵/기능 맵 요구를 충족한다.
- 우선순위와 범위가 명확하다.
- 관련 문서와 링크가 최신이다.
