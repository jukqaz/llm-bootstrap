# 스킬 릴리즈

## Overview

스킬 릴리즈 전 검증과 패키징을 수행하고 문서를 갱신하라. 수정은 승인 후에만 진행하라.

범위: 검증/패키징/변경 로그 갱신
비범위: 실제 배포/공유, 버전 태깅

## Workflow

### 1) 릴리즈 범위 확인
- 대상 스킬과 버전을 확인하라.
- 변경 로그 범위를 확인하라.

### 2) 검증
- `skill-validator` 기준으로 구조/프론트매터를 확인하라.
- 오류가 있으면 수정 제안 후 승인받아라.

### 3) 패키징
- `skill-packager` 흐름에 따라 `.skill` 파일을 생성하라.
- 결과 파일 경로를 기록하라.

### 4) 문서 갱신
- 필요 시 `release-docs`로 `CHANGELOG.md`와 `README.md`를 갱신하라.
- 변경 내용은 요약해서 기록하라.

### 5) 완료 안내
- 릴리즈 산출물과 다음 단계(배포/공유)를 안내하라.

## Output checklist

- 대상 스킬과 버전이 명시된다.
- 패키징 결과 파일 경로가 포함된다.
- 변경 로그가 최신이다.
