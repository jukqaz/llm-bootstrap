# QA/테스트

## Overview

QA/테스트 역할에서 필요한 산출물과 작업 흐름을 정리하라. 한국어로 작성하고 문서/작업 결과를 간결하게 요약하라.

범위: QA 운영/품질 기준, 테스트 체계 요약, 산출물 통합 관리
비범위: 상세 테스트 계획(test-orchestrator), 리그레션 체크리스트(regression-plan), 테스트 리포트(test-report)

## Workflow

### 1) 컨텍스트 수집
- AGENTS.md, README.md, PRD.md, TRD.md를 확인하라.
- 목표, 범위, 제약, 일정 정보를 정리하라.
- 정보가 부족하면 먼저 질문하라.

### 2) 산출물 정의
- 필요한 산출물을 QA 기준/체계 요약, 테스트 계획 연결, 리그레션 체크리스트 연결, 테스트 결과 요약 기준으로 정리하라.
- 산출물 우선순위를 정하고 범위를 명확히 하라.

### 3) 세부 스킬 연계
- 필요 시 다음 스킬을 사용하라:
  - 테스트 계획: test-orchestrator
  - 리그레션: regression-plan
  - 테스트 리포트: test-report
- 외부 도구나 MCP 설정이 필요하면 먼저 확인하라.

### 4) 정리 및 공유
- 결과를 핵심 요약과 함께 공유하라.
- 관련 문서/링크를 갱신하라.

## Output checklist

- 산출물이 역할 목표와 범위를 충족한다.
- 우선순위와 완료 기준이 명확하다.
- 관련 문서/링크가 최신이다.
