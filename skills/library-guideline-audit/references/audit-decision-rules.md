# Library Guideline Audit Rules

## Mismatch classification

- `유지 가능`
  - 현재 패턴이 공식 문서 예시와 다르지만 지원 범위 안에 있고 즉시 리스크가 없다.
- `권장 수정`
  - deprecated는 아니지만 현재 공식 권장 패턴과 차이가 커서 다음 수정 주기에 맞추는 것이 좋다.
- `교체 검토`
  - 라이브러리 유지 자체보다 대체 라이브러리의 공식 지원, 생태계, 유지보수성이 더 낫다.
- `즉시 수정 필요`
  - deprecated, 보안 이슈, 지원 중단, 명백한 오동작 위험이 있다.

## Replacement rationale template

- 현재 선택: 무엇을 쓰고 있는가
- 공식 기준: 현재 권장 경로가 무엇인가
- 문제점: 유지보수, API 안정성, 보안, 성능, 생태계 중 무엇이 걸리는가
- 유지안: 현재 라이브러리를 유지하면서 맞출 수 있는가
- 교체안: 교체 시 얻는 이점과 비용은 무엇인가
- 후속 작업: 테스트, CI, 문서, 운영 영향은 무엇인가
